◆ getConfigDirectory()

static std::string xy::FileSystem::getConfigDirectory ( const std::string &  appName)
static

Returns a path to the current user's config directory. Config files should generally be written to this directory, rather than the current working directory. Output is usually in the form of the following:

Windows: C:\Users\squidward\AppData\Roaming\appname\
Linux: /home/squidward/.config/appname/
Mac: /Users/squidward/Library/Application Support/appname/

WARNING some linux distros are known to return the current working directory instead. This should be considered when using configuration files with the same name as another which is expected to be stored in a unique directory.

Parameters
appNameName of the current application used to create the appname directory
Returns
Above formatted string, or an empty string if something went wrong
