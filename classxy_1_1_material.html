Detailed Description

Materials are used when rendering Model components. One or more materials can be assigned to a single Model to be applied to the Model's Mesh and SubMeshes. Materials contain, at the very least, an OpenGL 3.2+ compatible shader, as well as allowing specifying multiple properties (in the guise of uniform values) which are applied to the shader, such as textures or matrices. Materials may also have UniformBuffers added to them for convenience, and are required for the view/projection matrices at the very least when rendering with the MeshRenderer. The UniformBuffer for these (and the optional buffer for lighting properties) can be retrieved with MeshRenderer::getMatrixUniforms(). This should be the first thing to check if a material is apprently not renderering. The MaterialResource class is provided as a convenience for managing the lifetime of Materials as the are a shared resource. Bear also in mind that, as a shared resource, updating the property of a Material instance will affect ALL models to which thie Material is applied.
