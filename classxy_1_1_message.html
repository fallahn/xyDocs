Detailed Description

Message class.

The message class contains an ID used to identify the type of data contained in the message, and the message data itself. xygine uses some internal messaging types, so custom messages should have their IDs start at Message::Type::Count

TEMPLATE_GUARD const T& xy::Message::getData ( ) const
inline

Returns the actual data containend in the message.

Using the ID of the message to determine the data type of the message, this function will return a reference to that data. It is important to request the correct type of data from the message else behaviour will be undefined.

if(msg.id == Type::PhysicsMessage) { const PhysicsEvent& data = msg.getData<PhysicsEvent>(); do stuff with data }
