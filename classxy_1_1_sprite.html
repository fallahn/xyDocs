Detailed Description

Batched Sprite. While these sprites are slightly less flexible than SFML sprites (they do not carry their own transform and are bound entirely to that of their parent entity's) xy::Sprites belong to a parent batch which allows efficient drawing of multiple sprites in a single call potentially offering a large performance boost. A sprite component on its own will not work - it must be created via the addSprite() function of a SpriteBatch component.
